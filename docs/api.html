API

EventsManager​

Hyoga exposes an event-driven API through a global EventsManager instance, allowing the host application to communicate with player instances via publish/subscribe (pub/sub) pattern.

How it works​

When a Hyoga player is instantiated in the DOM, it reads the globaleventsmanager attribute (format: <uid>@<globalVariableName>) and creates (or reuses) a global window[globalVariableName] instance of EventsManager. This global variable exists only when at least one player instance is present in the DOM.

The EventsManager manages events organized by category and player id, supporting two categories:

CategoryPurpose
videoNative video and ad lifecycle events emitted by the player
lomaApplication-level commands and events for cross-component communication

API Methods​

sub(category, playerId, events, callback)​

Subscribe to one or more events.

ParamTypeDescription
categorystringEvent category: "video" or "loma"
playerIdstringThe player UID
eventsstring | string[]Event name(s) to subscribe to
callback(event, data) => voidHandler invoked with event name and payload

pub(category, playerId, event, data)​

Publish an event to all subscribers.

ParamTypeDescription
categorystringEvent category: "video" or "loma"
playerIdstringThe player UID
eventstringEvent name
dataobjectPayload object passed to subscribers

destroy(playerId, eventCategories)​

Remove all subscriptions for a given player.

ParamTypeDescription
playerIdstringThe player UID
eventCategoriesstring[]Categories to clean up, e.g. ['video', 'loma']

Configuration​

The globaleventsmanager attribute on the <hyoga-player> element defines the binding:

<hyoga-player globaleventsmanager="player001@lomaEventsManager" ...></hyoga-player>

This creates window.lomaEventsManager (if not already existing) and registers the player under id player001.

Subscribable Events (category: video)​

The player publishes all native video events and ad lifecycle events:

Native video events: loadstart, loadedmetadata, loadeddata, canplay, play, playing, pause, timeupdate, progress, seeking, seeked, waiting, durationchange, volumechange, ended

Player events: dispose, sourceset, play-request, sourceloaded

Ad events: ads-load, ads-request-init, ads-request-success, ads-request-error, ads-has-preroll, ads-no-preroll, ads-playprerolls, ads-endedprerolls, ads-playmidrolls, ads-endedmidrolls, ads-playpostrolls, ads-endedpostrolls, ads-pod-started, ads-pod-ended, ads-ad-started, ads-first-quartile, ads-midpoint, ads-third-quartile, ads-ad-ended, ads-play, ads-pause, ads-timeupdate, ads-click, ads-error

Application Events (category: loma)​

trigger-player-action​

Send playback commands to a player instance.

lomaEventsManager.pub("loma", uid, "trigger-player-action", { action: "play" });

Supported actions:

ActionDataDescription
play—Resume or start playback
pause—Pause playback
ffw{ timeDelta: number, forcePlay?: boolean }Fast-forward by timeDelta seconds
rwd{ timeDelta: number, forcePlay?: boolean }Rewind by timeDelta seconds
restart{ forcePlay?: boolean }Restart from beginning

loma-update-content​

Load new content into an existing player instance without re-mounting.

lomaEventsManager.pub("loma", uid, "loma-update-content", {
  endpoint: "https://sonic-endpoint.example.com",
  realm: "dmax",
  playbackType: "video",
  assetId: "video-12345",
  forcePlay: true
});
ParamTypeDescription
endpointstringSonic API endpoint
realmstringSonic realm
playbackTypestring"video" or "channel"
assetIdstringSonic asset/video ID
forcePlayboolean(optional) Auto-play after loading

Example: Subscribe to player events​

// Listen for video end
lomaEventsManager.sub("loma", "player001", ["ended"], (event, data) => {
  console.log("Video ended", data);
});

// Listen for source loaded
lomaEventsManager.sub("video", "player001", ["sourceloaded"], (event, data) => {
  console.log("Content metadata:", data.content);
});
caution

window.lomaEventsManager is only available after a <hyoga-player> element with a globaleventsmanager attribute has been connected to the DOM. Any attempt to use it before that will result in undefined.

globaleventsmanageryes—Format <uid>@<globalVarName> — creates/reuses window[globalVarName]

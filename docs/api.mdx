# API

## EventsManager

Hyoga exposes an **event-driven API** through a global `EventsManager` instance, allowing the host application to communicate with player instances via publish/subscribe (pub/sub) pattern.

### How it works

When a Hyoga player is instantiated in the DOM, it reads the `globaleventsmanager` attribute (format: `<uid>@<globalVariableName>`) and creates (or reuses) a global `window[globalVariableName]` instance of `EventsManager`. This global variable **exists only when at least one player instance is present in the DOM**.

The `EventsManager` manages events organized by **category** and **player id**, supporting two categories:

| Category | Purpose |
|----------|---------|
| `video`  | Native video and ad lifecycle events emitted by the player |
| `loma`   | Application-level commands and events for cross-component communication |

### API Methods

#### `sub(category, playerId, events, callback)`

Subscribe to one or more events.

| Param | Type | Description |
|-------|------|-------------|
| `category` | `string` | Event category: `"video"` or `"loma"` |
| `playerId` | `string` | The player UID |
| `events` | `string \| string[]` | Event name(s) to subscribe to |
| `callback` | `(event, data) => void` | Handler invoked with event name and payload |

#### `pub(category, playerId, event, data)`

Publish an event to all subscribers.

| Param | Type | Description |
|-------|------|-------------|
| `category` | `string` | Event category: `"video"` or `"loma"` |
| `playerId` | `string` | The player UID |
| `event` | `string` | Event name |
| `data` | `object` | Payload object passed to subscribers |

#### `destroy(playerId, eventCategories)`

Remove all subscriptions for a given player.

| Param | Type | Description |
|-------|------|-------------|
| `playerId` | `string` | The player UID |
| `eventCategories` | `string[]` | Categories to clean up, e.g. `['video', 'loma']` |

---

### Configuration

The `globaleventsmanager` attribute on the `<hyoga-player>` element defines the binding:

```html
<hyoga-player globaleventsmanager="player001@lomaEventsManager" ...></hyoga-player>
```

This creates `window.lomaEventsManager` (if not already existing) and registers the player under id `player001`.

---

### Subscribable Events (category: `video`)

The player publishes all native video events and ad lifecycle events:

**Native video events:** `loadstart`, `loadedmetadata`, `loadeddata`, `canplay`, `play`, `playing`, `pause`, `timeupdate`, `progress`, `seeking`, `seeked`, `waiting`, `durationchange`, `volumechange`, `ended`

**Player events:** `dispose`, `sourceset`, `play-request`, `sourceloaded`

**Ad events:** `ads-load`, `ads-request-init`, `ads-request-success`, `ads-request-error`, `ads-has-preroll`, `ads-no-preroll`, `ads-playprerolls`, `ads-endedprerolls`, `ads-playmidrolls`, `ads-endedmidrolls`, `ads-playpostrolls`, `ads-endedpostrolls`, `ads-pod-started`, `ads-pod-ended`, `ads-ad-started`, `ads-first-quartile`, `ads-midpoint`, `ads-third-quartile`, `ads-ad-ended`, `ads-play`, `ads-pause`, `ads-timeupdate`, `ads-click`, `ads-error`

---

### Application Events (category: `loma`)

#### `trigger-player-action`

Send playback commands to a player instance.

```js
lomaEventsManager.pub("loma", uid, "trigger-player-action", { action: "play" });
```

Supported actions:

| Action | Data | Description |
|--------|------|-------------|
| `play` | — | Resume or start playback |
| `pause` | — | Pause playback |
| `ffw` | `{ timeDelta: number, forcePlay?: boolean }` | Fast-forward by `timeDelta` seconds |
| `rwd` | `{ timeDelta: number, forcePlay?: boolean }` | Rewind by `timeDelta` seconds |
| `restart` | `{ forcePlay?: boolean }` | Restart from beginning |

#### `loma-update-content`

Load new content into an existing player instance without re-mounting.

```js
lomaEventsManager.pub("loma", uid, "loma-update-content", {
  endpoint: "https://sonic-endpoint.example.com",
  realm: "dmax",
  playbackType: "video",
  assetId: "video-12345",
  forcePlay: true
});
```

| Param | Type | Description |
|-------|------|-------------|
| `endpoint` | `string` | Sonic API endpoint |
| `realm` | `string` | Sonic realm |
| `playbackType` | `string` | `"video"` or `"channel"` |
| `assetId` | `string` | Sonic asset/video ID |
| `forcePlay` | `boolean` | (optional) Auto-play after loading |

---

### Example: Subscribe to player events

```js
// Listen for video end
lomaEventsManager.sub("loma", "player001", ["ended"], (event, data) => {
  console.log("Video ended", data);
});

// Listen for source loaded
lomaEventsManager.sub("video", "player001", ["sourceloaded"], (event, data) => {
  console.log("Content metadata:", data.content);
});
```

:::caution
`window.lomaEventsManager` is only available after a `<hyoga-player>` element with a [`globaleventsmanager`](guides/host-application-integration#core-attributes) attribute has been connected to the DOM. Any attempt to use it before that will result in `undefined`.
:::

## `<hyoga-player>` Attributes

The `<hyoga-player>` [Web Component](https://developer.mozilla.org/en-US/docs/Web/API/Web_components) accepts the following attributes to configure its behavior. For full integration examples and lifecycle details, see the [Integration guide](guides/host-application-integration#dom-setup).

### Core

| Attribute | Required | Default | Description |
|-----------|----------|---------|-------------|
| `id` | yes | — | Unique Hyoga manager identifier |
| `uid` | yes | — | Player UID, used as addressing key in all `pub`/`sub` calls |
| [`globaleventsmanager`](guides/host-application-integration#core-attributes) | yes | — | Format `<uid>@<globalVarName>` — creates/reuses `window[globalVarName]` |
| `playerselector` | no | `hyogaPlayer-<uid>` | CSS selector for the inner video element |
| [`videolibrary`](guides/host-application-integration#core-attributes) | no | `videojs` | Player engine: `dashjs`, `hlsjs`, or `videojs` |
| [`sourcetype`](guides/host-application-integration#core-attributes) | no | `sonic` | Content source: `sonic` or `direct` |
| [`playbacktype`](guides/host-application-integration#core-attributes) | no | `video` | `video` or `channel` |

### Sonic Source


| Attribute | Required | Default | Description |
|-----------|----------|---------|-------------|
| `endpoint` | yes | — | Sonic API endpoint URL |
| `realm` | yes | — | Sonic realm |
| `assetid` | yes | — | Sonic video/channel ID |
| `clientid` | no | `WEB:UNKNOWN:wbdatv:2.1.9` | Client identifier passed to Sonic |
| `discoparams` | no | `''` | Extra discovery parameters passed to Sonic |
| `token` | no | — | Initial authentication token |

### Direct Source


| Attribute | Required | Default | Description |
|-----------|----------|---------|-------------|
| `src` | yes | — | Direct video source URL |
| `srctype` | yes | — | Stream type (`dash`, `hls`) |
| `title` | no | — | Video title |
| `description` | no | — | Video description |
| `poster` | no | — | Poster image URL |

### Playback

| Attribute | Required | Default | Description |
|-----------|----------|---------|-------------|
| `streamtype` | no | — | Stream format hint (`dash`, `hls`); auto-detected from `videolibrary` if omitted |
| `autoplay` | no | `false` | Auto-start playback |
| `muted` | no | `false` | Start muted |
| `controls` | no | `true` | Show native Hyoga controls |
| `position` | no | — | Start position in milliseconds |
| `startover` | no | — | Enable start-over for live streams (HbbTV only) |
| `sourceparams` | no | — | Custom query params appended to the playback URL (HbbTV only) |
| `disableobserver` | no | `false` | Disable IntersectionObserver-based pause/resume |
| `disablepip` | no | `false` | Disable Picture-in-Picture |
| `representationsMax` | no | — | Maximum number of video representations |
| `representationsMin` | no | — | Minimum number of video representations |

### UI

| Attribute | Required | Default | Description |
|-----------|----------|---------|-------------|
| `hideoverlay` | no | `false` | Hide the default overlay |
| `locale` | no | `en` | Language code for UI strings |

### Advertising

| Attribute | Required | Default | Description |
|-----------|----------|---------|-------------|
| `adsystem` | no | `fw` | Ad system: `fw` (FreeWheel) or `ima` (Google IMA) |
| `adtagurl` | no | — | Ad tag URL (for `adsystem="ima"`) |
| `deferredadinit` | no | `false` | Defer ad request until user interaction |

### Content Filtering

| Attribute | Required | Default | Description |
|-----------|----------|---------|-------------|
| `maxagerating` | no | — | Maximum age rating for content filtering |
